# kvmdb
Key Value in Memory DataBase

Key Value In Memory DataBase на базе cowboy 2 и Json REST API с возможно подписки на обновления по всем ключам. Ожидаемое количество возможны ключей в базе - 1k (конфигурируемо), value может быть любым но максимальный размер до 100кб (конфигурируемо).

- Данные хранятся только в памяти, при попытке добавления ключей свыше указанного лимита необходимо возвращать ошибку;
- На GET сервер должен отдавать поток последних значений по kv в формате application/x-ld-json, пример подписки на стрим:
   `curl -s -N -H "Accept:application/x-ld-json" --compressed http://localhost:8181/test_api/v1/stream`.
   (Пояснение: при подключении отдаются все текущие значения, соединение сохраняется, при поступлении новых значений -
   обновления так же попадают в стрим)- Для медленных клиентов можно пропускать промежуточные обновление kv, но важно чтобы значение для каждого ключа было как можно более актуальным
Ожидается что к итоговой базе может быть подключено единовременно несколько сотен потребителей.
